# タスク管理アプリ 要件定義書

**作成日：** 2026年4月30日  
**バージョン：** 1.0  

---

## 1. プロジェクト概要

### アプリ名（仮）
**TaskBoard**（Trello風タスク管理アプリ）

### 目的
個人のタスクをボード形式で視覚的に管理する。  
カードをドラッグ&ドロップで移動し、作業の進捗を直感的に把握できるようにする。

### 利用者
- 自分ひとりで使用（マルチユーザー機能なし）
- 認証・ログイン機能なし

---

## 2. 動作環境

| 項目 | 内容 |
|------|------|
| 動作場所 | ブラウザ（Chrome推奨） |
| サーバー | 不要（ローカルのみ） |
| データ保存 | localStorage（ブラウザ内に永続保存） |
| ネット接続 | 不要 |

> ブラウザを閉じてもデータは消えません。

---

## 3. 使用技術（技術スタック）

| 役割 | ツール |
|------|--------|
| 画面作成 | React + TypeScript |
| デザイン | Tailwind CSS |
| ドラッグ&ドロップ | @dnd-kit |
| データ保存 | localStorage |
| プロジェクト構築 | Vite |

---

## 4. 画面構成

```
┌─────────────────────────────────────────┐
│  ヘッダー（アプリ名）                      │
├──────────┬──────────┬───────────────────┤
│ やること  │  進行中  │      完了          │

└─────────────────────────────────────────┘
```

## 5. 機能要件

### 5-1. ボード機能
| # | 機能 | 詳細 |
|---|------|------|
| B-1 | ボードの表示 | アプリ起動時にボードを1つ表示する |
| B-2 | ボードタイトルの編集 | ボード名をクリックして編集できる |

### 5-2. リスト機能
| # | 機能 | 詳細 |
|---|------|------|
| L-1 | リストの作成 | 「＋リストを追加」ボタンでリストを追加できる |
| L-2 | リストの削除 | リストを削除できる（確認ダイアログあり） |
| L-3 | リストタイトルの編集 | リスト名をクリックして編集できる |
| L-4 | 初期リスト | 初回起動時に「やること」「進行中」「完了」の3つを自動作成 |

### 5-3. カード機能
| # | 機能 | 詳細 |
|---|------|------|
| C-1 | カードの作成 | 「＋追加」ボタンでカードをリストに追加できる |
| C-2 | カードの削除 | カードを削除できる |
| C-3 | カードタイトルの編集 | カード名をクリックして編集できる |
| C-4 | カードの移動 | ドラッグ&ドロップでリスト間・リスト内を移動できる |
| C-5 | カードの詳細 | カードをクリックすると詳細モーダルが開く |
| C-6 | メモ欄 | カード詳細にメモ（自由記述）を追加できる |
| C-7 | 期日設定 | カードに期日を設定できる |
| C-8 | 期日の色分け | 期日が過ぎたカードは赤、今日は黄色で表示 |

### 5-4. データ保存機能
| # | 機能 | 詳細 |
|---|------|------|
| D-1 | 自動保存 | 変更のたびにlocalStorageへ自動保存 |
| D-2 | 初回データ | 初回起動時にサンプルデータを表示 |
| D-3 | 永続化 | ブラウザを閉じても・再起動してもデータが残る |

---

## 6. 非機能要件

| # | 項目 | 内容 |
|---|------|------|
| N-1 | レスポンシブ | PC・タブレットで見やすい表示 |
| N-2 | 操作性 | ドラッグ&ドロップが滑らかに動作する |
| N-3 | 表示速度 | 画面の読み込みが2秒以内 |

---

## 7. 対象外（今回作らないもの）

- ユーザー認証（ログイン・パスワード）
- 複数人での共有・リアルタイム同期
- スマートフォン対応
- ファイル添付機能
- 通知機能
- データのエクスポート（将来的に追加可能）

---

## 8. 開発ステップ（実装順）

```
ステップ1：プロジェクト作成（Vite + React + TypeScript）
ステップ2：データ構造の設計（ボード・リスト・カードの型定義）
ステップ3：UIの骨格作成（ヘッダー・ボード・リスト・カード）
ステップ4：カードのCRUD（追加・表示・編集・削除）
ステップ5：リストのCRUD
ステップ6：ドラッグ&ドロップ実装
ステップ7：localStorage保存
ステップ8：カード詳細モーダル（メモ・期日）
ステップ9：デザイン仕上げ（Tailwind CSS）
```

---

## 9. データ構造（イメージ）

```typescript
// ボード
Board {
  id: string
  title: string
  lists: List[]
}

// リスト（列）
List {
  id: string
  title: string
  cards: Card[]
}

// カード（タスク）
Card {
  id: string
  title: string
  memo: string       // メモ
  dueDate: string    // 期日
  createdAt: string  // 作成日
}
```
