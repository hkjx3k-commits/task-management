## 1. プロジェクト概要

Trello風のタスク管理アプリ。ボード・リスト・カードの3階層構造で個人タスクを管理する。  
サーバーレス・認証なし。データはlocalStorageに永続保存。

---

## 4. データ型定義

```typescript
// 優先度の種類
type Priority = "high" | "medium" | "low";

// カード（個々のタスク）
type Card = {
  id: string;
  title: string;
  memo: string;        // メモ（自由記述）
  dueDate: string;     // 期日（ISO 8601形式: "2026-05-10"）
  createdAt: string;   // 作成日時（ISO 8601形式）
  priority: Priority;  // 優先度（デフォルト: "medium"）
  order: number;       // リスト内での表示順（小さいほど上）
};

// リスト（列）
type List = {
  id: string;
  title: string;
  cards: Card[];
};

// ボード（全体）
type Board = {
  id: string;
  title: string;
  lists: List[];
};
```

## 5. ドラッグ&ドロップ設計

@dnd-kit を使用する。

```
DndContext（最上位）
└── SortableContext（各リスト）
    └── useSortable（各カード）
```

- **同一リスト内の移動**：`arrayMove` で並び替え → 変更後に `order` を 0, 1, 2… と振り直してlocalStorageへ保存
- **リスト間の移動**：`over` のIDを判定し、対象リストへカードを移動 → 移動先リストの `order` を振り直して保存

### 並び替えボタンの仕様

各リストのヘッダー付近に以下の2ボタンを配置する。

| ボタン | 動作 |
|--------|------|
| 優先度順 | そのリスト内のカードを `high → medium → low` の順に並び替え、`order` を振り直して保存 |
| 期日順 | そのリスト内のカードを `dueDate` の昇順（早い順）に並び替え。期日未設定のカードは末尾。`order` を振り直して保存 |

**重要：** 両ボタンともあくまで「その瞬間の一括整列」であり、並び替え後は通常のドラッグ&ドロップによる自由な並び替えが引き続き有効となる。ソート状態を維持するロック機能はない。

ステップ4：useLocalStorage カスタムフックの作成
ステップ5：Header・Board・List・Card コンポーネントの作成
ステップ6：カードのCRUD実装
ステップ7：リストのCRUD実装
ステップ8：@dnd-kit によるドラッグ&ドロップ実装
ステップ9：CardModal（詳細・メモ・期日・優先度）実装
ステップ10：期日の色分けロジック実装
